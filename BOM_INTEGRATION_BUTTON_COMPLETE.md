- ? Implemented `IntegrateBomWithLinesAsync` method following VBS script pattern
- ? Added Lines collection access (oLines)
- ? Multiple fallback methods for adding lines (nAddLine, nNew, nNewLine)
- ? Proper COM object cleanup
- ? Comprehensive error logging

**Key Method**:
```csharp
private async Task<bool> IntegrateBomWithLinesAsync(
    SageSessionService session, 
    BomImportBill bomHeader, 
    List<BomImportBill> bomLines)
```

**Steps** (mirrors VBS exactly):
1. Create `BM_Bill_bus` object
2. Set key with parent item code (`nSetKey`)
3. Call `nNew()` to create new BOM
4. Set header values (BillDesc1$, BillType$)
5. Access Lines collection (`oLines` or `Lines`)
6. For each line:
   - Add new line (`nAddLine`, `nNew`, or `nNewLine`)
   - Set ComponentItemCode$
   - Set QuantityPerBill$
   - Set optional fields (Reference$, CommentText$)
7. Write lines (if supported)
8. Write BOM (`nWrite`)
9. Clean up COM objects

## ?? Technical Details

### BM_Bill_bus vs Separate Objects

**Old Approach** (Removed):
```csharp
// Create header
var billHeader = session.CreateBusinessObject("BM_BillHeader_bus");
billHeader.nSetKey(parentItemCode);
billHeader.nWrite();

// Create each line separately
foreach (var line in lines)
{
    var billDetail = session.CreateBusinessObject("BM_BillDetail_bus");
    billDetail.nSetKeyValue("BillNo$", parentItemCode);
    billDetail.nSetKeyValue("ComponentItemCode$", componentCode);
    billDetail.nWrite();
}
```

**New Approach** (Implemented):
```csharp
// Create combined BOM object
var billBus = session.CreateBusinessObject("BM_Bill_bus");
billBus.nSetKey(parentItemCode);
billBus.nNew();

// Set header fields
billBus.nSetValue("BillDesc1$", description);
billBus.nSetValue("BillType$", "S");

// Access lines collection
var oLines = billBus.oLines; // or billBus.Lines

// Add all lines
foreach (var line in lines)
{
    oLines.nAddLine(); // or nNew() or nNewLine()
    oLines.nSetValue("ComponentItemCode$", componentCode);
    oLines.nSetValue("QuantityPerBill$", quantity);
}

// Write entire BOM (header + all lines) at once
billBus.nWrite();
```

**Advantages of New Approach**:
- ? Matches VBS reference script exactly
- ? Transactional (all or nothing)
- ? Fewer COM object creations
- ? Better performance
- ? Cleaner code

### Lines Collection Access

**Multiple Fallback Methods**:
```csharp
// Try different property names
dynamic? oLines = null;
try
{
    oLines = billBus.oLines;  // First attempt
}
catch
{
    try
    {
        oLines = billBus.Lines;  // Second attempt
    }
    catch
    {
        // Handle failure
    }
}
```

**Adding Lines - Multiple Methods**:
```csharp
int lineAdded = 0;
try
{
    lineAdded = oLines.nAddLine();  // Method 1
}
catch
{
    try
    {
        lineAdded = oLines.nNew();  // Method 2
    }
    catch
    {
        try
        {
            lineAdded = oLines.nNewLine();  // Method 3
        }
        catch
        {
            // All methods failed
        }
    }
}
```

**Why Multiple Methods?**:
- Different Sage versions use different method names
- Provides better compatibility
- Matches VBS script fallback logic

### COM Object Cleanup

**Proper Cleanup Pattern**:
```csharp
dynamic? oLines = null;
dynamic? billBus = null;

try
{
    // Use objects...
}
finally
{
    // Cleanup in reverse order of creation
    if (oLines != null)
    {
        try
        {
            oLines.DropObject();
            Marshal.ReleaseComObject(oLines);
        }
        catch (Exception ex)
        {
            _logger.LogWarning("Error releasing Lines object: {0}", ex.Message);
        }
    }

    if (billBus != null)
    {
        try
        {
            billBus.DropObject();
            Marshal.ReleaseComObject(billBus);
        }
        catch (Exception ex)
        {
            _logger.LogWarning("Error releasing BOM business object: {0}", ex.Message);
        }
    }
}
```

## ?? Validation Rules

### BOMs Ready for Integration

**Criteria**:
1. ? Status = "Validated"
2. ? All component items exist in Sage
3. ? Parent item exists in Sage
4. ? No new buy items required
5. ? No new make items required

**SQL Query**:
```sql
SELECT COUNT(DISTINCT ParentItemCode)
FROM isBOMImportBills
WHERE Status = 'Validated'
```

**Excluded from Integration**:
- ? Status = "NewBuyItem" (components need to be purchased first)
- ? Status = "NewMakeItem" (components need to be created first)
- ? Status = "Failed" (validation errors)
- ? Status = "Duplicate" (already exists)
- ? Status = "Integrated" (already integrated)

**2. Sage Connection Errors**:

- Parent item doesn't exist
- Component item doesn't exist
- Invalid quantities
