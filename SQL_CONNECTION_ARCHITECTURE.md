## Architecture Highlights

### Clean Architecture Layers
1. **Domain/Shared** - Interfaces and contracts (no dependencies)
2. **Application** - Models and business logic
3. **Infrastructure** - Concrete implementations (SQL, file I/O)
4. **UI** - WPF presentation layer

### Dependency Flow
```
UI ? Infrastructure ? Application ? Shared
UI ? Application ? Shared
```

### Separation of Concerns
- **SettingsService**: Manages file persistence
- **DatabaseConnectionService**: Handles SQL connections
- **SettingsViewModel**: UI logic and data binding
- **SettingsView**: Visual presentation

### Design Patterns Used
- **Dependency Injection**: All services injected via constructor
- **Repository Pattern**: Data access abstraction
- **MVVM Pattern**: Clear separation of UI and logic
- **Service Layer**: Business logic in services
