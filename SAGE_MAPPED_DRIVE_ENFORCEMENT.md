## ?? Overview

The Sage integration **requires** that the Sage Home directory be accessible via a **mapped drive letter** (e.g., `Z:\Sage\MAS90\Home`). Network paths (UNC paths like `\\server\share\Sage`) are **not supported** and will cause integration to fail.

This implementation adds validation and user warnings to ensure users configure Sage correctly.

---

## ?? Problem: Why Network Paths Fail

### UNC Path (? Not Supported)
```
\\server\share\Sage\MAS90\Home
```

**Why it fails:**
- COM objects (ProvideX.Script) cannot properly access network paths
- Authentication issues with network credentials
- Session context problems
- File locking issues across network

### Mapped Drive (? Supported)
```
Z:\Sage\MAS90\Home
```

**Why it works:**
- Drive letter acts as a local reference
- COM objects can access mapped drives
- Proper session context maintained
- Windows handles network authentication

## ?? Detection Logic

### 1. **UNC Path Detection**

```csharp
private bool IsUncPath(string path)
{
    if (string.IsNullOrWhiteSpace(path))
        return false;

    // UNC paths start with \\ or //
    return path.StartsWith(@"\\") || path.StartsWith("//");
}
```

**Detects:**
- `\\server\share\path`
- `//server/share/path`

**Returns:** `true` if network path, `false` otherwise

---

### 2. **Mapped Drive Detection**

```csharp
private bool IsMappedDrive(string path)
{
    if (string.IsNullOrWhiteSpace(path))
        return false;

    // Mapped drives start with a drive letter followed by colon (e.g., Z:, X:)
    if (path.Length >= 2 && char.IsLetter(path[0]) && path[1] == ':')
    {
        var driveLetter = path.Substring(0, 2);
        var driveInfo = new DriveInfo(driveLetter);
        
        // Network drives have DriveType.Network
        return driveInfo.DriveType == DriveType.Network;
    }

    return false;
}
```

**Detects:**
- `Z:\path` (network drive)
- `X:\path` (network drive)

**Differentiates from local drives:**
- `C:\path` (local drive) ? `false`
- `D:\path` (local drive) ? `false`

## ?? How to Map a Network Drive

### Windows Instructions

**Method 1: File Explorer**

```
1. Open File Explorer (Windows + E)
2. Click "This PC" in left sidebar
3. Click "Computer" tab ? "Map network drive"
4. Choose drive letter (e.g., Z:)
5. Folder: \\server\share\Sage
6. ? Check "Reconnect at sign-in"
7. Click "Finish"
```

**Method 2: Command Line**

```cmd
net use Z: \\server\share\Sage /persistent:yes
```

**Method 3: PowerShell**

```powershell
New-PSDrive -Name "Z" -PSProvider FileSystem -Root "\\server\share\Sage" -Persist
```
