## Key Components Summary

### 1. **ILoggerService Interface**
- Defines logging contract
- 5 log levels (Debug, Info, Warn, Error, Critical)
- Supports parameterized messages
- Exception support

### 2. **FileLoggerService Implementation**
- File-based logging
- Thread-safe with lock
- Automatic daily files
- Rotation at 10 MB
- Keeps 5 backup files

### 3. **Service Integration**
- DatabaseConnectionService
- SettingsService
- SettingsViewModel
- All services log operations

### 4. **Global Exception Handlers**
- AppDomain exceptions
- Dispatcher exceptions
- Task exceptions
- Never crashes application

### 5. **Log File Management**
- Location: `%APPDATA%\Aml.BOM.Import\Logs\`
- Format: `BomImport_YYYY-MM-DD.log`
- Automatic rotation
- Thread ID tracking
- Timestamp precision (milliseconds)

## Log Entry Lifecycle

```
1. Method Call
   ?
2. Logger.LogXxx(message, args)
   ?
3. Format message with parameters
   ?
4. Add timestamp, level, thread ID
   ?
5. Add exception details (if present)
   ?
6. Acquire lock (_lockObject)
   ?
7. Check file size
   ?
8. Rotate if needed (> 10 MB)
   ?
9. Write to file
   ?
10. Release lock
    ?
11. Return (never throws)
```
